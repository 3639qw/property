저는 AI와 법률 정보 시스템을 접목하는 데 깊은 관심을 가진 개발자 이주용입니다. 특히 복잡하고 일반인이 접근하기 어려운 정보를 기술을 통해 명료하게 만드는 것을 목표로 합니다. 곧 제가 복잡한 규칙과 시스템을 빠르게 이해하고 분석하는 데 뛰어난 집중력과 몰입도를 가지고 있음을 의미합니다.

이번 '부동산등기 챗봇' 프로젝트를 통해, 저는 단순한 코딩을 넘어 실제 사회 문제 해결에 기여하는 개발자가 되고자 합니다. 특히 상속등기와 관련된 방대한 법규, 예규, 선례를 RAG 모델에 효과적으로 학습시키는 과정에서 정보 구조화 및 자연어 처리 기술에 대한 깊이 있는 이해를 얻고 있습니다.

저는 프론트엔드(JS, HTML)와 백엔드(Python)를 아우르는 풀스택 역량을 기반으로, 사용자 친화적인 인터페이스와 견고한 백엔드 아키텍처를 결합하여 실용적인 애플리케이션을 구축하는 데 집중합니다. 복잡한 법률 문제를 AI로 풀어내는 이 프로젝트를 성공적으로 완수하여, AI가 일상생활에 기여하는 사례를 만들고자 합니다.

🚀 프로젝트 상세: 부동산등기 챗봇

📌 문제 정의

일반인이 부동산등기, 특히 상속등기에 관련된 복잡한 등기법, 호적법, 등기 예규 및 선례를 정확하게 찾고 이해하기가 매우 어려웠습니다. 등기관의 판단 기준이 되는 이러한 법률 자료들을 일반인의 눈높이에서 쉽게 접근하고 연관된 답변을 즉각적으로 얻을 수 있는 솔루션을 만들고 싶었습니다. 본 프로젝트는 이 정보 접근성 문제를 해결하는 것을 목표로 합니다.

🏛️ 아키텍처/설계

법정상속 및 협의분할상속에 관한 방대한 등기 예규와 선례를 데이터베이스화하고, 이를 RAG모델에 학습시켜 설계했습니다. 이 설계는 모델이 법규에 근거하여 답변하도록 하여, AI의 환각(Hallucination) 현상을 최소화하고 답변의 신뢰도를 높이는 데 중점을 두었습니다.

📚 학습 데이터

챗봇은 실제 등기 실무에서 활용되는 법령과 예규 문서를 기반으로 학습했습니다. 특히 상속등기와 관련하여 빈번하게 참조되는 공식 PDF 자료를 선별하여 RAG 모델의 지식 소스로 활용함으로써 답변의 신뢰성과 실무 적합성을 높였습니다.

  • 부동산등기법
  • 부동산등기규칙
  • 상속등기와 관련된 업무처리 지침
  • 대법원 등기선례, 등기예규
  • 가족관계등록법, 호적법
  • 민법, 의용민법

※ 위 문서들은 PDF 형태로 수집·정제하여 청크 단위로 분할 후 임베딩에 활용되었습니다.

🔑 핵심 기능: RAG 모델을 활용한 등기 법령의 학습

등기관이 상속등기에 관하여 법령, 기존 선례, 예규 등에 따라 등기를 교합하므로, 이 핵심 법률 자료들을 RAG 모델에 학습시켜 사용자의 질문에 대한 정확하고 근거 있는 답변을 제공합니다.

상속에 관하여 주요 쟁점이 되는 예규, 선례를 직접 조사하고 선별했습니다. 이를 효과적인 방법(예: 청크 사이즈 최적화, 임베딩 모델 선택)으로 RAG 모델에 학습시킬 방법을 강구하고 구현했습니다.

결과 및 성과

본 챗봇은 단순한 실험이 아니라, 실제 개인적인 필요에서 출발한 프로젝트입니다. 저는 실제로 상속등기를 직접 진행해야 하는 상황을 겪었고, 복잡한 법령·예규·관행을 이해하는 과정에서 상당한 시간과 시행착오를 경험했습니다. 이 과정에서 일반인이 스스로 해결하기에는 상속등기와 관련된 법률 정보의 접근 장벽이 매우 높다는 문제를 인식하게 되었고, “필요한 법률 정보를 즉시 물어볼 수 있는 챗봇이 있다면 도움이 되겠다”는 문제의식이 본 프로젝트의 출발점이 되었습니다.

아래는 실제 챗봇에 입력하여 테스트한 주요 질문 사례이며, 각 질문에 대해 챗봇은 학습된 법령, 등기 예규, 선례를 근거로 답변을 생성합니다.

  • 등기할 수 있는 권리의 종류 (소유권, 전세권, 지상권, 임차권 등)
  • 소유권이전등기를 신청할 때 첨부해야 할 문서 (계약서, 주민등록초본 등)
  • 상속등기를 신청할 때 첨부해야 할 문서 (호적등본, 상속재산분할협의서 등)
  • 등기명의인표시변경등기의 개념 및 적용 사례
  • 지상권의 존속기간 및 관련 법적 근거

이러한 질문에 대해 챗봇은 단순 요약이 아닌, 실제 등기관의 판단 기준이 되는 법령과 예규를 근거로 답변을 제시함으로써 정보의 신뢰성과 실무 활용 가능성을 높였습니다. 이를 통해 일반 사용자도 부동산등기 절차를 보다 체계적으로 이해할 수 있음을 확인했습니다.

배운 점 / 개선점

본 프로젝트를 통해 AI를 처음으로 본격 학습하며, 로컬 LLM 기반 법률 AI의 한계를 명확히 체감했습니다. 초기에는 로컬 모델을 활용하여 독립적인 시스템 구축을 시도했으나, 질문 1회당 응답 시간이 1분 이상 소요되는 등 실사용에는 무리가 있을 만큼의 문제가 발생했습니다. 이를 통해 법률과 같이 정확성과 응답 속도가 모두 중요한 영역에서는 로컬 모델보다 Gemini, ChatGPT와 같은 외부 API 기반 LLM을 활용하는 것이 현실적인 선택임을 깨닫게 되었습니다.

또한 AI의 답변 품질은 모델 자체뿐만 아니라 질문 구조와 학습 데이터의 품질에 크게 의존한다는 점을 배웠습니다. 실제로 AI가 사용자의 질문 의도를 정확히 이해하지 못하는 경우가 예상보다 많았으며, 이를 개선하기 위해서는 법률 문서의 정교한 구조화, 충분한 학습 데이터 확보, 그리고 프롬프트 설계에 대한 지속적인 실험과 노력이 필요함을 인식하게 되었습니다.

마지막으로 기능 구현에 집중한 결과 UI/UX 측면에서 개선의 여지가 있음을 확인했습니다. 향후에는 법률 지식이 없는 사용자도 자연스럽게 질문할 수 있도록 입력 방식과 화면 흐름을 보다 사용자 친화적으로 설계할 계획입니다.
